src.training.semi_supervised
============================

.. py:module:: src.training.semi_supervised

.. autoapi-nested-parse::

   FixMatch semi-supervised training for wafer defect detection.

   Leverages the ~640K unlabeled wafers in WM-811K alongside the ~172K labeled
   ones. High-confidence pseudo-labels on weakly-augmented unlabeled samples
   supervise strongly-augmented versions, yielding consistency regularization
   without external labeling effort.

   .. rubric:: References

   [111] Sohn et al. (2020). "FixMatch: Simplifying Semi-Supervised Learning
         with Consistency and Confidence". arXiv:2001.07685
   [114] Berthelot et al. (2019). "MixMatch: A Holistic Approach to
         Semi-Supervised Learning". arXiv:1905.02249

         arXiv:1905.03670

.. py:function:: get_weak_transform() -> torchvision.transforms.Compose

   Weak augmentation: horizontal flip only.

   FixMatch uses minimal augmentation for the "teacher" branch so that the
   pseudo-label is as clean as possible.


.. py:function:: get_strong_transform() -> torchvision.transforms.Compose

   Strong augmentation: flip + rotation + affine + noise + erasing.

   FixMatch uses aggressive augmentation for the "student" branch so that
   the model learns invariance to a wide range of perturbations.

.. py:class:: FixMatchTrainer(model: torch.nn.Module, num_classes: int = 9, confidence_threshold: float = 0.95, lambda_u: float = 1.0, device: str = 'cpu')

   FixMatch semi-supervised training for wafer defect detection.

   Uses labeled data with standard cross-entropy loss combined with unlabeled
   data via a pseudo-label consistency loss.  High-confidence predictions on
   weakly-augmented unlabeled samples become pseudo-labels for the
   corresponding strongly-augmented versions.

   .. attribute:: model

      The classification model being trained.

   .. attribute:: num_classes

      Number of target classes (9 for WM-811K).

   .. attribute:: confidence_threshold

      Minimum softmax probability to accept a
      pseudo-label (tau in the paper).

   .. attribute:: lambda_u

      Weight for the unsupervised consistency loss.

   .. attribute:: device

      Compute device string ("cpu" or "cuda").

   .. rubric:: References

   [114] Berthelot et al. (2019). "MixMatch". arXiv:1905.02249

   .. py:method:: train(labeled_loader: torch.utils.data.DataLoader, unlabeled_loader: torch.utils.data.DataLoader, val_loader: torch.utils.data.DataLoader, optimizer: torch.optim.Optimizer, criterion: torch.nn.Module, epochs: int = 50, scheduler: Optional[torch.optim.lr_scheduler.LRScheduler] = None, gradient_clip: Optional[float] = None, monitored_metric: str = 'val_macro_f1') -> Tuple[torch.nn.Module, Dict[str, Any]]

      Train with the FixMatch algorithm.

      Per iteration:
          1. Sample labeled batch ``(x_l, y_l)`` and unlabeled batch
             ``(x_u_weak, x_u_strong)``.
          2. Supervised loss: ``L_s = CE(model(x_l), y_l)``.
          3. Pseudo-labels: ``q = softmax(model(x_u_weak))``,
             ``mask = max(q, dim=1) >= threshold``.
          4. Unsupervised loss: ``L_u = CE(model(x_u_strong[mask]),
             argmax(q[mask]))``.
          5. Total: ``L = L_s + lambda_u * L_u``.

      :param labeled_loader: DataLoader yielding ``(images, labels)``.
      :param unlabeled_loader: DataLoader yielding ``(weak_images, strong_images)``.
      :param val_loader: DataLoader for validation (labeled).
      :param optimizer: Optimizer instance.
      :param criterion: Loss function for supervised branch.
      :param epochs: Number of training epochs.
      :param scheduler: Optional LR scheduler.
      :param gradient_clip: Optional max gradient norm.
      :param monitored_metric: Metric used for best-model selection.

      :returns: ``(best_model, history_dict)`` where *history_dict* contains
                per-epoch losses, accuracies, F1 scores, and pseudo-label
                statistics.



.. py:function:: extract_unlabeled_maps(df: pandas.DataFrame, known_classes: List[str]) -> List[numpy.ndarray]

   Extract wafer maps that are NOT in known_classes.

   The WM-811K dataset has ~811K total wafers but only ~172K have a failure
   type label in ``known_classes``. The remaining ~640K are either marked
   as empty/unknown or have no label at all. This function collects those
   unlabeled maps for use in semi-supervised training.

   :param df: Full WM-811K DataFrame with ``failureClass`` and ``waferMap`` columns.
   :param known_classes: List of valid label strings (e.g., KNOWN_CLASSES).

   :returns: List of raw numpy wafer-map arrays (variable sizes).
